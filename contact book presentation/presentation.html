How the program works

The data of each user
[John, Lark, 19830505, 1337, 35, 505, taurus]

They are then stored in a master array
[<John's data>, <Nuode's data>, [...], [...]...]

Each of that user's attributes is inserted into its respective attribute tree (AVL tree) as the new node's key,
the corresponding value of the node will be the index of user's data in giant array

If an array is used, wouldn't deletion be O(n)?

Introducing the   .nuode 
file format.

All the contact book data is securely stored in database.nuode, which can only be opened from the inside..

It contains:

  • > Date when contact book was last opened
  • > Data of the user itself
  • > Data of the all the friends in a giant array
  • > A string representations of ADTs used
    1. AVL trees
    2. Stacks
    3. Nested custom arrays
